# where-to-go 使用说明

> 一款《洛克王国：世界》辅助地图工具 —— 单文件 `where-to-go.exe`，无需安装。

## 一、功能概览

- **完整大地图**：拖拽 / 缩放 / 三图层切换（地表 G / 地下 B1 / B2）/ 全部官方点位。
- **路径规划**：手动连线、自由绘制、TSP 自动连接最近邻；路径可保存、命名、按图层归档。
- **实时定位**：框选游戏右上角小地图后，每 ~600ms 截屏 + NCC 模板匹配，把玩家位置 + 朝向同步到大地图。
- **路径导航**：把已有路径设为活动路径，已走段灰色、未走段亮绿描边并嵌入流动白色箭头；玩家投影点处自动圆滑过渡。
- **位置预测**（可开关）：在两次 NCC 匹配之间按最近速度外推显示位置，画面更跟手。
- **游戏内悬浮窗**：透明置顶窗口，叠加在游戏之上，可调大小 / 透明度，可一键鼠标穿透。

## 二、首次运行

1. 双击 `where-to-go.exe` 启动。
2. 首次会进入 **资源加载页**，自动从 `wiki.biligame.com/rocom/大地图` 下载：
   - 地图元数据、点位类别、点位坐标、文本图层、区域边界
   - 默认缩放级别 (z=4) 的全部地图瓦片
3. 加载完成后进入主界面，后续启动直接使用本地缓存。
4. **缓存目录**：`<exe 同级目录>/cache/`；exe 所在目录不可写时自动落到 `%LOCALAPPDATA%\where-to-go\cache\`。
5. **配置文件**：`<exe 同级目录>/config.json`（标记样式、图标尺寸、ROI、追踪 / 预测开关、悬浮窗参数等）。

> 高缩放（更清晰）的瓦片在你放大具体区域时按需下载并永久缓存，无需手动操作。

## 三、主界面操作

| 操作 | 方式 |
|---|---|
| 平移地图 | 中键拖动 / 平移工具下左键拖动 |
| 缩放 | 鼠标滚轮 |
| 切换图层 | 左侧栏 "图层" 区点 G / B1 / B2 |
| 显示/隐藏点位类别 | 左侧栏勾选类别 |
| 框选 | 左键拖动（在选择工具下） |
| 右键菜单 | 在地图任意位置右键，可加点 / 命名 / 删除 |
| 撤销 | `Ctrl+Z`（地图焦点下） |
| 重置视图 | 右下角 "回到原点" 按钮 |

## 四、路径规划

1. 工具栏切到 **链接 / 自由绘制 / TSP** 之一。
2. 在地图上依次点击或拖动产出路径点。
3. 用 **TSP 自动连接** 让程序基于点位图用 K-NN + 2-opt 求近似最短回路；用 **链接** 工具点两点用 A\* 自动布线。
4. **保存草稿** 后输入路径名称，写入 `<exe 同级目录>/data/routes/*.json`。
5. 侧栏 "路径" 区可勾选 / 隐藏 / 重命名 / 删除。

## 五、定位 + 路径导航

1. 启动游戏，调整到固定的小地图（右上角，**北锁定**）。
2. 主界面或菜单点 **"框选小地图"**（默认热键 `Ctrl+Alt+R`）。屏幕变暗后按住左键画矩形覆盖游戏小地图圆形区域，松开确认。
3. 主界面 **"开始追踪"**：每 ~600ms 截一次该区域，识别出玩家世界坐标 + 朝向，在大地图上以蓝色圆 + 绿色三角箭头显示。
4. 选中一条已保存路径并 **"开始导航"**，匹配频率提升到 ~400ms；路径会按当前进度自动分成 "已走 / 未走" 两段；自动居中（可在设置中改成 "off / always / navonly"）。
5. **位置预测**：勾选 "启用位置预测" 后，两次 NCC 匹配之间按最近 1.5 秒内的速度外推位置，缓解可见的滞后；速度低于 ~2 单位/秒（基本静止）时不外推。

## 六、悬浮窗

| 操作 | 默认热键 |
|---|---|
| 显示 / 隐藏悬浮窗 | `Ctrl+Alt+M` |
| 切换鼠标穿透 | `Ctrl+Alt+T` |
| 开始/停止追踪 | `Ctrl+Alt+S` |
| 框选小地图 | `Ctrl+Alt+R` |

悬浮窗显示与主窗口完全一致的 **地图本体**（无侧栏 / 工具栏），共享同一个 MapView 状态：在主窗口选中路径、切换图层、缩放，悬浮窗同步刷新。窗口可拖动 / 缩放（边缘）/ 调整透明度；穿透模式下窗口忽略所有鼠标事件，可正常游戏。

## 七、性能与精度小贴士

- **大量点位**：开启所有类别勾选会显著拖慢渲染。仅勾选当前需要的类别（地图渲染层只画在视口内的点位与路径段，但开点位太多仍会有 GPU 压力）。
- **超长路径**：保存的路径节点数 > 200 时，所有视口外的折线段 / 节点编号 / 箭头 / 流动动画都会被剔除，性能开销可控。
- **小地图识别不准**：确认游戏小地图设置为 *固定北朝向*；框选时尽量贴边、不要圈进圆框外的边角。
- **悬浮窗在某些游戏全屏下不显示**：将游戏切换为 *无边框窗口* 模式即可。

## 八、命令行参数

```text
where-to-go.exe                  正常启动
where-to-go.exe -reset           清空缓存并重新抓取
where-to-go.exe -no-fetch        即使无缓存也不联网
where-to-go.exe -cmd fetch       命令行模式只跑一次抓取
where-to-go.exe -cmd test-recog  用 testMinimapImg 跑识别自检
where-to-go.exe -cmd test-overlay 仅打开悬浮窗供调试（验证渲染管线）
where-to-go.exe -cmd test-hotkey 仅注册热键并打印事件
```

## 九、常见问题

- **首次加载卡住 / 失败**：检查能否访问 `wiki-dev-patch-oss.oss-cn-hangzhou.aliyuncs.com`，可用 `-reset` 重试。
- **悬浮窗顶部出现白边**：已修复（Win32 NCCALCSIZE 抑制非客户区）。如仍出现，请确认运行的是最新版本。
- **悬浮窗打开后开启路径显示崩溃**：已修复（MapView 共享状态加锁）。
- **导航时已走 / 未走线粗细不一致**：已修复（统一双层管道样式 + 节点圆形过渡）。
